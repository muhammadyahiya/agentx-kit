# agentx-kit v1.4.0 — Release Notes

**Git tag:** `v1.4.0`

---

## Highlights

- **New: `agentx.azure`** — a production Azure AI platform layer, in the same
  spirit as the [enterprise pack](../features/enterprise.md):
    - 8 service wrappers (Blob, Service Bus, Cosmos DB, Key Vault, Event Grid,
      Container Apps, Azure ML, Monitor), each Managed-Identity-first with
      structured logging, correlation IDs, and retries baked in.
    - A fluent `AzurePipeline` builder — `.blob_storage(...).service_bus(...)...deploy()`
      — with a plan-before-apply default (`deploy(execute=True)` to actually wire it up).
    - Pre-wired templates: `AIOpsPipeline`, `MLOpsPipeline`, `RAGPipeline` (built
      on `agentx.rag`), `ChatbotPipeline`, `DocumentAIPipeline`.
    - A new `agentx azure` CLI (`init`/`create`/`plan`/`deploy`/`destroy`) + YAML
      pipeline config.
    - Extras-gated per service (`azure-blob`, `azure-servicebus`, …, bundled as
      `azure-platform`) — every wrapper lazy-imports its SDK only inside its
      constructor, so `agentx azure --help` works with zero Azure SDK packages
      installed.
    - See [docs/features/azure.md](../features/azure.md) and
      [docs/cli/azure.md](../cli/azure.md).
- **New: repo CI** (`.github/workflows/ci.yml`) — ruff + ty + the full pytest
  suite (incl. `azure-platform`) now run on every push/PR; previously only
  `publish.yml` (tag-triggered PyPI publish) and `docs.yml` existed, so nothing
  gated a broken change from merging.
- **Fix**: `agentx version` reported `1.1.0` while the package itself was
  already at `1.3.0` (`src/agentx/__init__.py.__version__` had drifted from
  `pyproject.toml`) — both are back in sync.

## Upgrading

```bash
pip install -U agentx-kit
pip install -U "agentx-kit[azure-platform]"   # to use agentx.azure
```

No breaking changes — this is an additive release.
